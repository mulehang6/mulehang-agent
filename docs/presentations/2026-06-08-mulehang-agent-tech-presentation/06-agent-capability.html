Exploration Flow

简单代码探索流程

当前 agent 可以围绕文件系统做简单探索。典型任务是看目录结构、读取指定文件，再把结果通过事件流反馈到 CLI。

一次简单探索会发生什么

  • 用户输入一个和代码库相关的问题。
  • Koog agent 判断需要先看目录还是先读文件。
  • tool registry 调用 `__list_directory__` 或 `__read_file__`。
  • 结果回到 runtime，再通过 SSE 发给 CLI。

当前适合演示的任务

  • 查看某个目录里有哪些模块。
  • 读取某个 Kotlin 或 TypeScript 文件。
  • 定位一个简单功能对应的代码位置。

User Prompt

询问代码库中的某个问题

List Directory

Read File

读取关键文件内容

Tool Call Event

把调用结果推回 CLI

这页要讲清楚的点

  • 当前 agent 的探索能力主要依赖文件工具。
  • CLI 会持续展示探索过程事件，用户可以看到完整过程。
  • 这条链路已经有实现和测试支撑，比较适合课堂展示。
