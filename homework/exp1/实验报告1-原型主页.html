本页面用于展示项目在实验一中的原型化表达方式。它不模拟完整 Web 产品， 而是把运行时核心结构压缩为一个静态主页，用于说明系统的输入、运行时主轴、 provider 与 binding、agent 能力接入以及统一输出模型。

核心功能分区

输入入口层

负责接收外部请求，并将其导入统一 runtime 主轴。当前主入口为 CLI，后续入口也只能复用同一套契约。

  • CLI 请求输入
  • 后续入口适配
  • 统一进入 runtime

Runtime 主轴

负责组织系统调用流，明确请求如何进入、如何流动、如何结束。

  • Session
  • Request Context
  • Capability Request
  • Error Boundary

Provider 与 Binding

负责把用户输入的连接信息转化为运行时真正可消费的模型 binding。

  • Custom Provider Profile
  • Provider Type
  • Connection Probe Result
  • Model Discovery Result
  • Executor / Model Binding

Agent 与 Capability Integration

基于 Koog 执行请求，并统一接入多种能力来源。

  • Koog AIAgent
  • Local / Custom Tools
  • MCP
  • Direct HTTP Internal API

一次请求的运行时流转

STEP 01 接收输入

入口层接收用户请求，并准备交给 runtime 处理。

STEP 02 建立上下文

runtime 创建 session、request context，并确定本次执行边界。

STEP 03 解析 binding

系统选择 provider、模型 binding 以及可用 capability set。

STEP 04 Agent 执行

Koog AIAgent 发起执行，并按需要调用工具、MCP 或 HTTP 适配能力。

STEP 05 统一输出

最终返回 RuntimeEvent、RuntimeResult 或 RuntimeFailure。

说明：该 HTML 原型只用于表达系统的运行时结构与核心流程，不包含真实请求执行、 配置保存、模型探测和工具调用逻辑。
